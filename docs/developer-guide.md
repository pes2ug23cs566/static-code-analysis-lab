# Developer Guide for Simple Subscription Billing Simulator

## Introduction
This document serves as a guide for developers who wish to contribute to the Simple Subscription Billing Simulator project. It outlines the setup process, coding standards, and contribution guidelines.

## Getting Started

### Prerequisites
Before you begin, ensure you have the following installed:
- Node.js (version X.X.X or higher)
- npm (Node package manager)
- TypeScript (version X.X.X or higher)

### Cloning the Repository
To get a local copy of the project, clone the repository using the following command:
```bash
git clone https://github.com/pestechnology/PESU_EC_CSE_I_P39_Simple_Subscription_Billing_Simulator_Uncharted.git
cd PESU_EC_CSE_I_P39_Simple_Subscription_Billing_Simulator_Uncharted
```

### Installing Dependencies
Once you have cloned the repository, install the necessary dependencies:
```bash
npm install
```

### Running the Application
To run the application, use the following command:
```bash
npm start
```

### Running Tests
To ensure that everything is working correctly, run the tests using:
```bash
npm test
```


## Coding Standards
- Follow the established coding conventions for JavaScript/TypeScript.
- Use meaningful variable and function names.
- Write comments to explain complex logic.
- Ensure your code is clean and well-organized.

## Branching Strategy
- Use `develop` for ongoing development.
- Create feature branches from `develop` for new features.
- Use `bugfix/*` branches for bug fixes.

## Commit Messages
Follow the conventional commit format:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for code style changes
- `refactor:` for code refactoring
- `test:` for test-related changes

## Code Review Process
1. Create a feature branch from `develop`.
2. Make your changes and commit them.
3. Create a Pull Request to `develop`.
4. Request a review from team members.
5. Merge after approval.

## Conclusion
Thank you for contributing to the Simple Subscription Billing Simulator project! Your efforts help improve the project and provide a better experience for users. If you have any questions, feel free to reach out to the team.
